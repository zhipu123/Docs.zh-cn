## <a name="conclusion"></a><span data-ttu-id="9e612-303">结束语</span><span class="sxs-lookup"><span data-stu-id="9e612-303">Conclusion</span></span>

 <span data-ttu-id="9e612-304">从其开始 Katana 项目的目标尚未创建和从而迫使开发人员了解另一个 Web 框架。</span><span class="sxs-lookup"><span data-stu-id="9e612-304">From its beginning, the Katana project's goal has not been to create and thereby force developers to learn yet another Web framework.</span></span> <span data-ttu-id="9e612-305">相反，目标已被创建为.NET Web 应用程序开发人员提供更多的选择不是以前曾出现可能的抽象。</span><span class="sxs-lookup"><span data-stu-id="9e612-305">Rather, the goal has been to create an abstraction to give .NET Web application developers more choice than has previously been possible.</span></span> <span data-ttu-id="9e612-306">来分割成一组可替换的组件的典型 Web 应用程序堆栈的逻辑层，Katana 项目使整个堆栈以提高在任何速率适合这些组件的组件。</span><span class="sxs-lookup"><span data-stu-id="9e612-306">By breaking up the logical layers of a typical Web application stack into a set of replaceable components, the Katana project enables components throughout the stack to improve at whatever rate makes sense for those components.</span></span> <span data-ttu-id="9e612-307">通过构建简单的 OWIN 抽象周围的所有组件，Katana 使框架和基于它们的应用程序可以跨各种不同的服务器和主机移植。</span><span class="sxs-lookup"><span data-stu-id="9e612-307">By building all components around the simple OWIN abstraction, Katana enables frameworks and the applications built on top of them to be portable across a variety of different servers and hosts.</span></span> <span data-ttu-id="9e612-308">通过将开发人员放入控件的堆栈，Katana 可确保开发人员进行有关如何轻量的最终选择或其 Web 堆栈应为如何功能丰富。</span><span class="sxs-lookup"><span data-stu-id="9e612-308">By putting the developer in control of the stack, Katana ensures that the developer makes the ultimate choice about how lightweight or how feature-rich her Web stack should be.</span></span>
